OLMo-core GRPO + OPD

Student rollouts stay online. A teacher vLLM scorer evaluates those exact student prefixes. DistillKit turns sparse teacher top-k logprobs into a forward-KL token loss that can augment GRPO or run by itself as pure OPD.

Student rollout Teacher scorer DistillKit loss Learner update

Where It Hooks

OPD enters after rollout/reward collection and before packing. This keeps one batch contract for baseline GRPO, GRPO + OPD, and pure OPD.

Teacher Top-k

For each response token, teacher vLLM returns the top K next-token ids and logprobs under the prefix the student actually visited.

Missing Mass

The loss omits teacher probability outside top-k by default. opd/teacher_topk_mass tells us whether top-k is large enough.

Pure OPD

Pure OPD still samples online from the student. It simply zeros the GRPO reward/policy loss and trains only on teacher KL.

GRPO + OPD

Use when: we still want task reward optimization.

task_loss = grpo_pg_loss + beta * ref_kl
total_loss = task_loss + opd_loss_coef * opd_loss

Typical knobs:
  --opd_enabled true
  --opd_use_task_rewards true
  --beta 0.0
  --load_ref_policy false

Pure OPD

Use when: we want online distillation only.

task_loss = 0
total_loss = opd_loss_coef * opd_loss

Typical knobs:
  --opd_enabled true
  --opd_use_task_rewards false
  --beta 0.0
  --load_ref_policy false

Validated Rehearsal

Qwen3-0.6B student + Qwen3-4B teacher, top-k 128, 500 steps, 2 GPUs, W&B tracking, final checkpoint saved. The run showed the whole path working: teacher loads, live scoring happens, tensors reach the learner, OPD loss logs, and training completes.

  • 1Teacher top-k mass was effectively 1.0, so top-k 128 captured almost all teacher mass.
  • 2Sampled-token-in-top-k was near 1.0, so teacher support covered student behavior.
  • 3This proves the system path, not yet OPD quality versus a matched GRPO-only control.

Mental model: OPD is online dense supervision. GRPO answers “did the final answer get reward?” OPD answers “at every visited prefix, what would the teacher distribution have preferred?”
